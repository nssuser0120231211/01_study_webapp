🏠 HTMLヘッダー要素ガイド

<head>要素に記述できる全ての要素を詳しく解説

🎯 ヘッダー要素とは？

<head>要素は、ページのメタデータ（ページに関する情報）を格納する領域です。ここに記述された内容は通常、ブラウザの画面には直接表示されませんが、ページの動作や検索エンジンでの表示に重要な役割を果たします。

📄 <title> - ページタイトル

ページのタイトルを定義します。ブラウザのタブやブックマーク、検索結果に表示されます。

<title>私のウェブサイト - ホームページ</title>
重要： titleタグは必須要素で、1つのページに1つだけ記述できます。SEOにも大きく影響します。
🏷️ <meta> - メタデータ

ページに関する様々な情報を提供する最重要要素です。

主要なmeta要素：

種類 記述例 用途
文字エンコーディング <meta charset="UTF-8"> 文字コードの指定（必須）
ビューポート <meta name="viewport" content="width=device-width, initial-scale=1.0"> レスポンシブデザイン対応
説明文 <meta name="description" content="ページの説明文"> 検索結果に表示される説明
キーワード <meta name="keywords" content="HTML, CSS, JavaScript"> 検索キーワード（現在は効果薄）
著者 <meta name="author" content="田中太郎"> ページの作成者
リフレッシュ <meta http-equiv="refresh" content="30"> 自動リロード設定

SNS共有用のOpen Graph（OG）タグ：

<meta property="og:title" content="ページタイトル"> <meta property="og:description" content="ページの説明"> <meta property="og:image" content="画像のURL"> <meta property="og:url" content="ページのURL"> <meta property="og:type" content="website">
🎨 <link> - 外部リソースの読み込み

CSS、アイコン、フォントなどの外部リソースを読み込みます。

用途 記述例
CSSファイル <link rel="stylesheet" href="style.css">
ファビコン <link rel="icon" href="favicon.ico">
Webフォント <link rel="preconnect" href="https://fonts.googleapis.com">
正規URL <link rel="canonical" href="https://example.com/page">
RSS <link rel="alternate" type="application/rss+xml" href="rss.xml">
前読み込み <link rel="preload" href="main.css" as="style">
📱 <style> - 内部CSS

HTMLファイル内にCSSスタイルを直接記述します。

<style> body { background-color: #f0f0f0; font-family: Arial, sans-serif; } .header { background: linear-gradient(45deg, #667eea, #764ba2); color: white; padding: 20px; } </style>
使い分け： 小規模なプロジェクトや特定ページのみのスタイルに適用。大規模なサイトでは外部CSSファイルを推奨。
⚡ <script> - JavaScript

JavaScriptコードや外部JavaScriptファイルを読み込みます。

内部JavaScript：

<script> console.log('ページが読み込まれました'); document.addEventListener('DOMContentLoaded', function() { // ページ読み込み完了後の処理 }); </script>

外部JavaScript：

<script src="script.js"></script> <script src="https://cdn.example.com/library.js"></script>

script要素の属性：

属性 説明 例
defer HTML解析完了後に実行 <script defer src="script.js"></script>
async 非同期でダウンロード・実行 <script async src="analytics.js"></script>
type スクリプトの種類 <script type="module" src="module.js"></script>
🔧 <base> - ベースURL

相対URLの基準となるベースURLを設定します。

<base href="https://example.com/subfolder/">

この設定により、相対パス image.jpg は https://example.com/subfolder/image.jpg として解釈されます。

注意： baseタグは1ページに1つまでで、できるだけhead要素の先頭に配置することを推奨。

🔍 SEO最適化のためのhead要素

<!-- 基本的なSEO要素 --> <title>具体的で魅力的なページタイトル（60文字以内推奨）</title> <meta name="description" content="ページの内容を分かりやすく説明（160文字以内推奨）"> <!-- 正規URL指定 --> <link rel="canonical" href="https://example.com/this-page"> <!-- ロボットの制御 --> <meta name="robots" content="index, follow"> <!-- 構造化データ（JSON-LD形式） --> <script type="application/ld+json"> { "@context": "https://schema.org", "@type": "Article", "headline": "記事のタイトル", "author": "著者名" } </script>

📱 PWA（Progressive Web App）対応

<!-- マニフェストファイル --> <link rel="manifest" href="manifest.json"> <!-- iOS用のメタタグ --> <meta name="apple-mobile-web-app-capable" content="yes"> <meta name="apple-mobile-web-app-status-bar-style" content="default"> <meta name="apple-mobile-web-app-title" content="アプリ名"> <!-- アイコン設定 --> <link rel="apple-touch-icon" sizes="180x180" href="icon-180x180.png"> <link rel="icon" type="image/png" sizes="32x32" href="icon-32x32.png">

💡 ベストプラクティス

  • 必須要素： charset、viewport、titleは必ず含める
  • 読み込み順序： metaタグ → title → CSS → JavaScript の順番を推奨
  • パフォーマンス： 重要なCSSは内部スタイル、大きなファイルは外部参照
  • SEO： titleとdescriptionは各ページで固有の内容にする
  • セキュリティ： CSP（Content Security Policy）の設定も検討
