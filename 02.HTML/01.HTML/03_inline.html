🧱 インライン要素 vs ブロック要素

HTMLの基本的な要素の分類と表示特性を理解しよう

📖 基本概念の理解

HTMLの要素は、その表示特性によって主にインライン要素とブロック要素に分類されます。この分類は、要素がページ上でどのように配置され、どのような空間を占有するかを決定します。

🔸 インライン要素

  • 横に並んで表示される
  • 内容の幅だけを占有
  • 改行されない
  • 上下のmargin/paddingが効かない
  • 他の要素と同じ行に配置可能

🔹 ブロック要素

  • 縦に積み重なって表示
  • 親要素の幅全体を占有
  • 前後で改行される
  • 全方向のmargin/paddingが有効
  • 高さと幅を指定可能

⚠️ よくある間違いと注意点

❌ 避けるべき間違い：

  • インライン要素にブロック要素を含める
    <!-- ❌ 間違い --> <span> <div>これは無効なHTML</div> </span>
  • インライン要素にwidth/heightを指定
    <!-- ❌ 効果がない --> <span style="width: 200px; height: 100px;">サイズは変わらない</span>

✅ 正しい使い方：

<!-- ✅ 正しい --> <div> <p>段落の中に <span>インライン要素</span> を配置</p> </div> <!-- ✅ 正しい --> <span style="margin: 0 10px; padding: 5px;">左右マージンとパディング</span>

🆕 現代のCSS：display プロパティによる制御

HTML5以降、要素の表示特性はCSSのdisplayプロパティで柔軟に制御できます：

/* インライン要素をブロック要素として表示 */ span { display: block; width: 200px; height: 100px; } /* ブロック要素をインライン要素として表示 */ div { display: inline; } /* インライン・ブロック（両方の特性を持つ） */ .inline-block { display: inline-block; width: 150px; height: 50px; margin: 10px; } /* フレックスボックスによるレイアウト */ .flex-container { display: flex; gap: 10px; } /* グリッドレイアウト */ .grid-container { display: grid; grid-template-columns: repeat(3, 1fr); }

現代の開発では、HTML要素の本来の意味（セマンティクス）を重視し、表示方法はCSSで制御するのがベストプラクティスです。

🏗️ レイアウト構造の構築：

<header> <nav> <ul> <li><a href="#">ホーム</a></li> <li><a href="#">サービス</a></li> <li><a href="#">お問い合わせ</a></li> </ul> </nav> </header> <main> <section> <h2>セクションタイトル</h2> <p>コンテンツの説明文</p> </section> </main>

💡 現代的なアプローチ（CSS Grid + Flexbox）：

<div class="grid-container"> <div class="card"> <h3>カードタイトル</h3> <p>カードの内容</p> </div> </div> <style> .grid-container { display: grid; grid-template-columns: repeat(auto-fit, minmax(300px, 1fr)); gap: 20px; } .card { display: flex; flex-direction: column; padding: 20px; border: 1px solid #ccc; border-radius: 8px; } </style>
